Introdução

WebDAV (Web-based Distributed Authoring and Versioning) permite que você carregue ou descarregue arquivos grandes, vários arquivos ou pastas de arquivos para o seu curso ou site do projeto com mais facilidade.

Antes de configurar o WebDAV, um arquivo ou pasta deve estar no destino da ferramenta Escaninho. Para obter instruções sobre como adicionar arquivos sem usar o WebDAV, consulte Adicionar, editar ou deletar um item na Escaninho.

Salvar arquivos diretamente para WebDAV (isto é, dentro de um aplicativo, como o Word, Excel ou Internet Explorer) não funciona de forma confiável. Para melhores resultados, salve o arquivo em seu computador primeiro, e depois faça o upload para o seu site.

Nota: Nomes de arquivos, que incluem determinados caracteres não são suportados. Para obter uma lista de caracteres não suportados, consulte Caracteres não suportados.

Usando Windows Vista

Nota: Conexão com recursos do seu site usando WebDAV em um computador que executa o Windows Vista não funciona de forma confiável. Os desenvolvedores estão atualmente investigando esta questão. No entanto, a Microsoft tem uma atualização de software disponível que pode ajudar a corrigir problemas de conexão para alguns usuários. Para obter informações sobre a atualização e como fazer o download, consulte o artigo 907306 na Ajuda e Suporte da Microsoft Web site.

Pesquisa de Ajuda e Suporte da Microsoft.

Para adicionar um atalho para o seu computador com Windows Vista que lhe permite carregar e acessar arquivos em Escaninho usando WebDAV:

  1. Na área de trabalho, clique em Computador.

  2. Clique em Mapear Unidade de Rede.

  3. Na tela Mapear Unidade de Rede, clique em Conectar a um site da Web para você armazenar seus documentos e imagens, e clique em Próximo.

  4. Clique em Escolher uma localização de rede personalizada, e clique em Próximo.

  5. No campo "Endereço de internet ou rede:", insira a URL que aponta para a ferramenta Repositório de destino.

    Contate o administrador do site para o formato da URL. Ele provavelmente incluirá o siteID de seu site.

    Nota: Para instruções sobre como achar seu siteID, veja Determinar o siteID de seu site.

Você pode agora arrastar e soltar pastas entre seu computador e os sites que você acabou de conectar. Para deletar um item, clique com o botão direito e selecione Excluir.

Usando Windows XP

No Windows XP, você pode utilizar o Assistente para Adicionar Local de Rede ou o Internet Explorer para carregar e acessar seus arquivos via WebDAV, se você tiver as permissões necessárias.

  1. >Na área de trabalho, clique duas vezes em Meus Locais de Rede.

  2. No painel "Tarefas de Rede", clique em Adicionar um local de rede.

  3. Na tela de boas-vindas, clique em Próximo.

  4. Selecione Escolher outro local de rede , e clique em Próximo.

  5. No campo "Endereço de internet ou rede:", insira a URL que aponta para a ferramenta Repositório de destino.

    Contate o administrador do site para o formato da URL. Ele provavelmente incluirá o siteID de seu site.

    Nota: Para instruções sobre como achar seu siteID, veja Determinar o siteID de seu site.

Você pode agora arrastar e soltar pastas entre seu computador e os sites que você acabou de conectar. No entanto, você não pode apagar itens de um site arrastando-o para a Lixeira em sua área de trabalho. Para apagar um item, clique com o botão direito e selecione Excluir.

Mac OS X 10.4 (Tiger) e 10.5 (Leopard)

Para usar o WebDAV para carregar e acessar arquivos no Mac OS X 10.4 (Tiger) e 10.5 (Leopard):

  1. No Localizador, pressione Command-k . (Alternativamente, a partir do menu Vá, escolha Conectar no Servidor.)

  2. No campo "Endereço de internet ou rede:", insira a URL que aponta para a ferramenta Repositório de destino.

    Contate o administrador do site para o formato da URL. Ele provavelmente incluirá o siteID de seu site.

    Nota: Para instruções sobre como achar seu siteID, veja Determinar o siteID de seu site.

Agora você pode arrastar e soltar pastas entre o computador e os sites que você acabou de conectar.

Opcionalmente, você pode clicar no + (sinal mais) à direita do endereço do servidor, e o servidor será adicionado à sua lista de servidores favoritos.

Quando você entra, você pode optar por lembrar a frase-chave. Se você fizer isso, não será solicitada uma senha quando você faz logon em seus recursos pessoais ou em recursos para qualquer site ao qual você tem acesso. Não faça isso a menos que seu computador esteja protegido contra o uso de outros.

Nota: Quando você utilizar o WebDAV para acessar recursos usando esse método, você pode perceber que os arquivos .DS_store são criados. Base de conhecimento da Apple fornece instruções para eliminar este comportamento.

Mac OS X 10.3 (Panther) e anteriores

Para usar o WebDAV para carregar e acessar arquivos no Mac OS X 10.3 (Panther) e anteriores requer um cliente de terceiros chamado Goliath.

Nota: Goliath tem alguns problemas de desempenho conhecidos, e pode não funcionar 100% do tempo. Devido a isso, apenas suporte limitado está disponível se você tiver problemas.

Você pode baixar uma cópia gratuita do Goliath em:

http://www.webdav.org/goliath/installs/Goliath1_0_1.dmg.gz

Depois de ter baixado o arquivo, clique duas vezes nele para descompactá-lo. Em seguida, clique duas vezes no arquivo .dmg. Uma imagem de disco chamada Goliath 1.0.1 agora deve estar disponível. então:

  1. Arraste e solte a pasta Goliath contida dentro do imagem de disco para disco rígido.

  2. Abra a aplicação Goliath.

  3. No campo "Endereço de internet ou rede:", insira a URL que aponta para a ferramenta Repositório de destino.

    Contate o administrador do site para o formato da URL. Ele provavelmente incluirá o siteID de seu site.

    Nota: Para instruções sobre como achar seu siteID, veja Determinar o siteID de seu site.

A primeira vez que você tentar se conectar, poderá ser solicitado a aceitar um certificado para o site. Clique em OK se for solicitado.

Agora você pode arrastar e soltar pastas entre o computador e os sites a que você acabou de conectar.
